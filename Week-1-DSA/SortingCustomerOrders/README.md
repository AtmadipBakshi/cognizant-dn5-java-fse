# Sorting Customer Orders

## Objective
Sort customer orders by total price to prioritize high-value orders.

---

## Sorting Algorithms Used

### Bubble Sort
- Repeatedly swaps adjacent elements
- Simple but inefficient

Time Complexity:
- Best: O(n)
- Average: O(n²)
- Worst: O(n²)

---

### Quick Sort
- Uses divide and conquer
- Picks a pivot and partitions array

Time Complexity:
- Best: O(n log n)
- Average: O(n log n)
- Worst: O(n²)

---

## Comparison

| Algorithm | Performance | Usage |
|----------|-------------|------|
| Bubble Sort | Slow | Educational |
| Quick Sort | Fast | Real-world systems |

---

## Conclusion
Quick Sort is preferred because it is significantly faster for large datasets compared to Bubble Sort.
